# US1 – Ask FAQ

## User Story
As a student,  
I want to submit my question through a simple interface,  
So that I can receive relevant answers about university information.

---

## Acceptance Criteria

### AC1 – Submit a valid question
- **Given** I am on the FAQ page,  
- **When** I type a valid question and click the “Submit” button,  
- **Then** the system should store my question and display a success confirmation message.

---

### AC2 – Required field validation
- **Given** I leave the question input empty,  
- **When** I try to submit,  
- **Then** the system should display an error message indicating that the field is required.

---

### AC3 – Edge Case: Excessive question length
- **Given** I enter a question that exceeds 300 characters,  
- **When** I try to submit,  
- **Then** the system should prevent submission and display a “Question too long” message.

---

## Definition of Done (DoD)
- [x] All acceptance criteria are met and tested  
- [x] Form validation is implemented (required & max length)  
- [x] Code reviewed, merged, and deployed to staging  
- [x] No blocker/major bugs remain  
- [x] PO/Stakeholder accepted the story
